# Инструкция по синхронизации (Локальный запуск)

Так как ваш проект запущен локально, автоматическая синхронизация (которая обычно работает в облаке Vercel) отключена. Чтобы данные в OKK были актуальными, нужно запускать синхронизацию вручную.

## Как запустить синхронизацию

Я подготовил специальный скрипт `local-sync.sh` в корне проекта.

### Способ 1: Через терминал (Рекомендуется)
1. Откройте терминал в папке проекта.
2. Запустите команду:
   ```bash
   bash local-sync.sh
   ```

### Способ 2: Через браузер
Вы можете по очереди открыть эти ссылки в браузере (при запущенном проекте):
1. **История (Статусы):** [http://localhost:3000/api/sync/history](http://localhost:3000/api/sync/history) — это обновит статусы заказов.
2. **Звонки:** [http://localhost:3000/api/sync/telphin](http://localhost:3000/api/sync/telphin) — это загрузит новые записи.
3. **Матчинг:** [http://localhost:3000/api/match](http://localhost:3000/api/match) — это привяжет звонки к заказам.

---

## Что делает скрипт синхронизации
- Подтягивает изменения статусов из RetailCRM.
- Обновляет таблицу заказов (Active Orders).
- Загружает новые звонки из Телфин.
- Сопоставляет звонки с заказами для анализа.

> [!TIP]
> Запускайте этот скрипт каждый раз, когда хотите увидеть актуальную статистику на дашборде после изменений в RetailCRM.
