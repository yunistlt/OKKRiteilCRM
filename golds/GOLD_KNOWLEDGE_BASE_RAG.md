# GOLD: Архитектура Базы Знаний и ИИ-Консультанта (RAG)

Этот документ определяет "Золотые правила" реализации многопользовательской базы знаний (Knowledge Base) и Retrieval-Augmented Generation (RAG) для ИИ-консультанта.

## 1. Архитектура Базы Данных (Multi-tenancy)
* **Единая таблица (Коллекция):** Все данные всех клиентов хранятся в одной базе данных (или векторном хранилище). ЗАПРЕЩАЕТСЯ создавать отдельные таблицы под каждого клиента.
* **Строгая изоляция (Row-Level Security):** В каждой таблице базы знаний обязано присутствовать поле `tenant_id` (или `client_id`). Все SQL/векторные запросы к базе ДОЛЖНЫ содержать фильтр `WHERE tenant_id = 'X'`, чтобы исключить утечку данных между клиентами.
* **Структура хранения:** 
  * `kb_documents`: Хранит полные исходные тексты статей и метаданные (название, источник).
  * `kb_embeddings`: Хранит разбитые на куски (чанки) тексты и их векторные представления (embeddings) для поиска.

## 2. Борьба с галлюцинациями ИИ
* **Системный Промпт (System Prompt):** Консультант обязан получать строгую инструкцию: отвечать ТОЛЬКО на основе предоставленного контекста. Запрещено додумывать факты, цены или условия.
* **Fallback-ответ:** Если ответа нет в контексте, модель обязана отвечать заготовленной фразой: "В моей базе знаний нет информации об этом. Пожалуйста, свяжитесь с оператором".
* **Temperature:** Устанавливаем параметр `temperature` (креативность модели) на значение `0` или `0.1` для обеспечения предсказуемости и точности.
* **Цитирование (Citations):** Ответы ИИ должны содержать ссылки на исходные документы из базы знаний, чтобы пользователь мог проверить источник информации.

## 3. Стратегия Поиска (Retrieval)
* **Гибридный Поиск (Hybrid Search):** Для получения лучшего контекста используем комбинацию:
  1. *Векторный поиск* (поиск по смыслу и синонимам).
  2. *Полнотекстовый поиск* (Keyword search / BM25) для точного нахождения артикулов, тарифов, специфичных терминов.
* **Чанкинг (Chunking):** Тексты клиентов должны нарезаться на смысловые куски (абзацы), а не произвольно посередине предложений.

## 4. UI/UX Наполнения Базы Знаний Клиентом
* **Гарантия чистоты данных:** Интерфейс должен подталкивать пользователя писать короткими, четкими правилами и инструкциями.
* **Встроенный тестер:** Клиент должен иметь возможность задать вопрос своему консультанту сразу после сохранения статьи, чтобы проверить, как алгоритм усвоил новую информацию.
