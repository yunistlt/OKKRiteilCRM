# Документация по VoC Knowledge Base

## Модерация кластеров и вопросов
- Все новые вопросы и кластеры проходят ручную модерацию через Supabase Studio или административный интерфейс.
- Модератор может:
  - Изменять intent_slug, category, question_variants, answer_website, answer_consultant
  - Менять статус is_active, frequency_score, type, tags
  - Удалять или архивировать нерелевантные записи
- Рекомендуется проводить ревью новых кластеров раз в неделю.

## Обновление базы знаний без даунтайма
- Для обновления базы знаний используйте сидер (scripts/seed_knowledge_base.ts) или Supabase Studio.
- Все изменения применяются атомарно (через транзакции или staging-таблицы).
- Для крупных обновлений:
  - Подготовьте новую версию данных в staging-таблице
  - Проверьте корректность через тесты и предпросмотр
  - Перенесите данные в основную таблицу knowledge_base_qa
- Для rollback используйте резервные копии или staging-таблицы.

## Контроль прогресса наполнения
- Используйте административный интерфейс для визуализации количества вопросов, претензий, замечаний, частотности и статуса модерации.
- Для автоматизации контроля можно добавить cron-отчёты и алерты в Telegram.
