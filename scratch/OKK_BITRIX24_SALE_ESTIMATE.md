## 2. Что нужно адаптировать под Bitrix24

### 2.1 Замена RetailCRM → Bitrix24 (ОСНОВНАЯ РАБОТА)

Текущая система глубоко интегрирована с RetailCRM через `syncOrderFromRetailCRM()`.  
Нужно переписать весь слой синхронизации данных.

| Задача | Детали | Оценка, ч |
|--------|--------|-----------|
| Маппинг сущностей | Сделки Bitrix24 → модель `orders` (статусы, поля, кастомные поля) | 20–30 |
| Синхронизация заказов | `syncOrderFromBitrix24()` через REST API Bitrix24 | 30–50 |
| Вебхуки Bitrix24 | Входящие события: смена статуса, новые сделки | 20–30 |
| История событий заказа | Маппинг активностей Bitrix24 → `raw_order_events` | 20–30 |
| Данные по менеджерам | Bitrix24 Users → таблица менеджеров системы | 15–20 |
| **Итого: CRM интеграция** | | **105–160 ч** |

### 2.2 Телефония (КРИТИЧЕСКИЙ РИСК)

Система написана под **Telphin** (таблица `raw_telphin_calls`).  
Результат зависит от того, что использует клиент:

| Вариант телефонии клиента | Объём работ | Оценка, ч |
|--------------------------|-------------|-----------|
| Telphin (тот же провайдер) | Только конфигурация | 5–10 |
| Встроенная телефония Bitrix24 | Новый адаптер через Activity/Call API | 60–100 |
| Mango Office / UIS / SIPUNI | Новый адаптер (webhooks + записи) | 50–80 |
| Другая АТС с API | Оценивается отдельно | 60–120 |

> ⚠️ **Важно уточнить у клиента ДО заключения договора.**  
> Телефония = самый большой риск по объёму.

### 2.3 Аутентификация и RBAC

| Вариант | Описание | Оценка, ч |
|---------|---------|-----------|
| Своя авторизация (Supabase Auth) | Создаём пользователей вручную, Bitrix24 = отдельно | 10–15 |
| SSO через Bitrix24 OAuth | Единый вход через Bitrix24-аккаунты | 40–60 |

> Рекомендую **SSO** — это значительно удобнее для клиента.

## 6. Варианты модели продажи

### Вариант A: Разовая продажа + поддержка
- Клиент платит фиксированную сумму за интеграцию
- Далее — ежемесячный абонемент на поддержку: **50 000–100 000 ₽/мес**
- Подходит для зрелых клиентов, которые берут на себя ИТ

### Вариант B: SaaS-аренда
- Клиент платит ежемесячно за доступ к системе
- Вы берёте на себя хостинг и обновления
- Стартовая цена: **80 000–150 000 ₽/мес**
- Окупаемость разработки: **12–18 месяцев**

### Вариант C: Белая метка (White Label)
- Клиент получает исходный код и право развёртывать самостоятельно
- Разовый платёж: базовая стоимость **× 1.5–2**
- Нет обязательств с вашей стороны после сдачи

## 8. Риски и условия

| Риск | Влияние | Митигация |
|------|---------|-----------|
| Телефония не поддерживает API записей | +200–400 ч | Уточнить до договора |
| Bitrix24 кастомные поля сложной структуры | +50–100 ч | Аудит CRM клиента до начала работ |
| On-premise требование | +100–200 ч | Уточнить на этапе пресейла |
| Нестандартная логика скоринга для их бизнеса | +80–150 ч | Фиксировать в ТЗ как отдельный этап |

## 9. Рекомендуемый формат договора

1. **Этап 0 (оплата 50%):** Аудит Bitrix24 клиента, согласование ТЗ, демо-доступ

3. **Этап 2 (оплата 20%):** UAT, устранение замечаний, продакшн-запуск
4. Гарантийный период: **3 месяца** (баги, не выявленные на UAT)
