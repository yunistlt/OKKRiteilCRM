# Спецификации виджета ОКК (Елена)

## 1. Функциональные возможности

### Консультация и база знаний
- **H2H (Human to Hybrid) общение:** Ответы на вопросы пользователя с использованием GPT-4o-mini.
- **Интеграция с БЗ:** Поиск технических характеристик (размеры, материалы, вес) в таблице `product_knowledge`.
- **Мультимодальность:** Возможность парсинга ссылок и анализа контекста текущей страницы.

### Сбор данных и лидогенерация
- **Детекция контактов:** Автоматическое извлечение телефона, email и ников Telegram из диалога.
- **Приём файлов:** Возможность загрузки спецификаций и ТЗ пользователем.
- **Передача Семёну:** Формирование структурированного объекта для Агента «Семён» для автоматического создания заказа в RetailCRM.

### Управление контекстом
- **История диалогов:** Сохранение веток обсуждений (threads) для возврата к контексту.
- **Привязка к заказу:** Если пользователь авторизован или идентифицирован, подтягивание статуса его текущего заказа.

## 2. Технические требования

### Frontend (Widget)
- **Изоляция стилей:** Использование Shadow DOM или строгих префиксов (как в текущем CSS `#okk-chat-widget`), чтобы стили сайта не ломали виджет.
- **Адаптивность:** Полноценная работа на mobile (fullscreen mode) и desktop (floating panel).
- **Состояние:** Сохранение состояния свернут/развернут и истории сообщений в `localStorage`.
- **Производительность:** Ленивая загрузка основного скрипта виджета, чтобы не блокировать LCP основного сайта.

### API и Backend
- **Streaming:** Поддержка потоковой передачи ответа (Server-Sent Events) для уменьшения Time to First Token.
- **Безопасность:** 
  - Rate limiting по IP (защита от перерасхода токенов).
  - Валидация входящих файлов на стороне сервера.
- **Интеграция:** Endpoint `/api/okk/consultant` должен принимать `threadId` для продолжения беседы.
