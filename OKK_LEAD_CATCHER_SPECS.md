# Спецификации модуля «ОКК: Ловец лидов» (Елена)

## 1. Функциональные возможности

### Консультация и база знаний
- **H2H (Human to Hybrid) общение:** Ответы на вопросы пользователя с использованием GPT-4o-mini.
- **Интеграция с БЗ:** Поиск технических характеристик (размеры, материалы, вес) в таблице `product_knowledge`.
- **Мультимодальность:** Возможность парсинга ссылок и анализа контекста текущей страницы.

### Сбор данных и лидогенерация
- **Детекция контактов:** Автоматическое извлечение телефона, email и ников Telegram из диалога.
- **Приём файлов:** Возможность загрузки спецификаций и ТЗ пользователем.
- **Передача Семёну:** Формирование структурированного объекта для Агента «Семён» для автоматического создания заказа в RetailCRM.

### Управление контекстом
- **История диалогов:** Сохранение веток обсуждений (threads) для возврата к контексту.
- **Привязка к заказу:** Если пользователь авторизован или идентифицирован, подтягивание статуса его текущего заказа.

## 2. Технические требования

### Frontend (Widget)
- **Изоляция стилей:** Использование Shadow DOM или строгих префиксов (как в текущем CSS `#okk-chat-widget`), чтобы стили сайта не ломали виджет.
- **Адаптивность:** Полноценная работа на mobile (fullscreen mode) и desktop (floating panel).
- **Состояние:** Сохранение состояния свернут/развернут и истории сообщений в `localStorage`.
- **Производительность:** Ленивая загрузка основного скрипта виджета, чтобы не блокировать LCP основного сайта.

### API и Backend
- **Streaming:** Поддержка потоковой передачи ответа (Server-Sent Events) для уменьшения Time to First Token.
- **Безопасность:** 
  - Rate limiting по IP (защита от перерасхода токенов).
  - Валидация входящих файлов на стороне сервера.
- **Интеграция:** Endpoint `/api/okk/consultant` должен принимать `threadId` для продолжения беседы.

### Интеграция и Callback-функции
- **Событийная модель:** Виджет должен поддерживать систему колбеков для интеграции с внешней аналитикой или логикой сайта.
- **onMessage:** Вызывается при получении нового сообщения от ИИ или отправке пользователем.
- **onLeadCaptured:** Срабатывает в момент, когда Елена успешно распознала контактные данные (телефон/email) в диалоге.
- **onWidgetToggle:** Уведомление о раскрытии или сворачивании окна чата.
### 3. Реализация и деплой

- **Интеграция через Webasyst:** Виджет реализован на стороне админки Webasyst через специализированный «блок» `OKK_chat`.
- **Управление блоком:** Это позволяет контент-менеджеру включать/выключать чат на конкретных витринах или страницах без правки исходного кода шаблонов.
- **Передача параметров:** Блок автоматически пробрасывает необходимые метаданные (ID текущего пользователя Webasyst, если он авторизован) в инициализацию виджета для бесшовной склейки контекста.
## 4. Модуль «Ловец Лидов» (Lead Catcher)

### 🕵️‍♂️ Назначение раздела
В системе ОКК выделен специализированный раздел **«Ловец Лидов»**, который служит центральным хабом для мониторинга активности Елены (Продуктолога) на внешних витринах.
- **Фиксация диалогов:** Все переписки пользователей с виджетом на сайте автоматически сохраняются и отображаются в реальном времени.
- **Прозрачность:** Менеджеры могут видеть, на каком этапе находится консультация и какие данные уже были запрошены ИИ.
- **Контроль качества:** Максим (Аудитор) может оценивать корректность ответов Елены и актуальность данных в базе знаний.

### 🗄️ Структура данных (Database)
Для обеспечения работы модуля в базу данных добавлены следующие таблицы:
- `widget_messages`: Хранит полную историю сообщений, включая метаданные (URL страницы, User-Agent, IP).
- `widget_threads`: Группирует сообщения в сессии, отслеживает статус идентификации клиента и привязку к RetailCRM.
- `lead_extractions`: Таблица-лог работы Семёна, где фиксируются извлеченные сущности (телефоны, email, ТЗ) до момента создания заказа.

### 🔄 Процесс передачи
1. **Елена** ведет диалог и помечает найденные контакты.
2. Данные попадают в `widget_messages` с флагом `has_lead_data`.
3. **Семён** раз в 1 минуту сканирует новые записи и инициирует создание лида/заказа в CRM, обновляя статус в `widget_threads`
