## Принципы работы

### Терминология
- **Чеклист** — жёсткие проверки, портят % рейтинга, встроены в код
- **Правила** — динамический контроль, создают Alerts/Violations, настраиваются через UI

### Интеграция данных
- Каждый агент специализируется на одной части pipeline
- Агенты передают результаты друг другу через таблицы и события
- Все действия логируются в audit и metrics

### Качество контроля
- Все выходные данные проходят validation
- Есть fallback-логика на сбой интегрирующей системы
- Rate limiting и лимиты по API-использованию
