# Reactivation (Виктория - Реактивационный контур)

**Статус:** ✅ Реализован в production

## Описание

Полнофункциональный контур для реактивации неактивных клиентов с четырьмя специализированными ролями Виктории:

### Роли Виктории

| Роль | Функция | Описание |
|------|---------|---------|
| **Виктория-Разведчик** | Scout (Collector) | Фильтрует клиентов по гибким критериям (давность, LTV, средний чек) |
| **Виктория-Писатель** | Writer | Пишет персональные неагрессивные письма от имени менеджера |
| **Виктория-Аналитик** | Analyst | Классифицирует ответы клиентов (POSITIVE/NEGATIVE/NEUTRAL) |
| **Виктория-Ответчик** | Responder | Пишет живые ответы на положительные ответы |

### Интеграция в ОКК

Результаты работы Виктории видны в основном дашборде ОКК:

| Статус | Иконка | Значение |
|--------|--------|---------|
| Письмо отправлено | ✉️ | Письмо успешно отправлено клиенту |
| Письмо прочитано | 👁️ | Клиент открыл и прочитал письмо (tracking pixel) |
| Ответ получен | 💬 | Клиент прислал ответ (тональность 🔥 hot или 🚫 cold) |

### Технологический стек

- **Scout:** API RetailCRM v5 (фильтрация по полям)
- **Writer:** GPT-4o-mini (с промптом из карточки кампании)
- **Analyst:** GPT-4o-mini (JSON mode, фиксированный промпт)
- **Responder:** GPT-4o-mini (персонализированный ответ)
- **Tracking:** Invisible 1x1 pixel в письмах для фиксации прочтений

## Статус реализации

- ✅ **Виктория-Разведчик** — работает, фильтрует по custom fields
- ✅ **Виктория-Писатель** — работает, персональные письма
- ✅ **Виктория-Аналитик** — работает, классификация ответов
- ✅ **Виктория-Ответчик** — работает (если enabled в кампании)

## Кампании

Администратор создаёт кампании с:
- Фильтрами для Scout (когда в последний раз был заказ, LTV, и т.д.)
- Промптом для Writer (стиль письма, предложение)
- Настройкой Responder (`on_positive: send_reply` или `create_order`)
- Расписанием отправки
