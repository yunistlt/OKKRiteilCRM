# AI-секретарь на входящие звонки

Голосовой секретарь принимает входящий звонок, проводит первичную квалификацию и переводит
на менеджера: по текущему заказу — на его менеджера, по новому запросу — создаёт заявку в
RetailCRM и переводит на наименее загруженного менеджера.

## Разделение ответственности

- **Телфин («AI секретарь»)** — голос: приём звонка, распознавание речи, синтез (TTS), перевод.
  Настраивается в конструкторе схем Телфина (Маршрутизация и номера), без кода.
- **Наша система** — «мозг»: один эндпоинт, который Телфин дёргает узлом «Интерактивная обработка».

Своего STT/TTS-стека не требуется.

## Поток

```
Входящий → AI секретарь: «Вы по текущему заказу или новая заявка?»
  ├─ по текущему заказу → донабор номера заказа (DTMF)
  │     → Интерактивная обработка → /api/telphin/secretary?mode=existing
  │     → находим заказ по номеру → менеджер → SetVar добавочный + TTS → перевод
  └─ новая заявка → AI секретарь квалифицирует (STT)
        → Интерактивная обработка → /api/telphin/secretary?mode=new
        → createSecretaryLead (статус всегда «Новая» novyi-1) + выбор менеджера по нагрузке
        → SetVar добавочный + TTS «Заявка создана» → перевод
```

Ветка (`existing`/`new`) задаётся нашим параметром `?mode=` в URL, который прописывается в схеме
Телфина для каждой ветки отдельно.

## Контракт «Интерактивной обработки» (Telphin Call Interactive)

**Запрос** (метод GET/POST — настраивается в схеме). Используемые параметры от Телфина:
- `CallerIDNum` — номер звонящего (АОН);
- `voice_navigator_DTMF` — набранные цифры (номер заказа в ветке existing);
- `voice_navigator_STT` — распознанная речь (суть запроса в ветке new);
- `CallID` — id звонка.

Плюс наши: `token` (авторизация), `mode` (ветка).

**Ответ** — XML `<Response>` с действиями (до 10; `hangup`/`SimpleTransfer`/`TTS` терминальны):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <SetVar name="transferDestination">106</SetVar>
  <TTS lang="ru-RU" voice="alena" play_now="true">Соединяю вас с менеджером Юлия Петрова.</TTS>
</Response>
```
После узла «Интерактивная обработка» в схеме должен идти узел **«Перевод на номер из
переменной»** с именем переменной `transferDestination` (см. `TELPHIN_TRANSFER_VAR`).

## Код

- `app/api/telphin/secretary/route.ts` — эндпоинт (GET+POST, XML-ответ, защита `?token=`).
- `lib/secretary/core.ts` — `findOrderByNumber`, `getManagerById`, `pickLeastLoadedManager`.
- `lib/secretary/telphin-xml.ts` — построение XML-ответа.
- `lib/retailcrm/leads.ts` → `createSecretaryLead` — создание заявки (novyi-1) + назначение менеджера.
- `middleware.ts` — `/api/telphin` в публичных префиксах.
- `app/settings/managers` — поле «Доб. Телфин» (добавочный менеджера).
- `migrations/20260628_telphin_secretary.sql` — `managers.telphin_extension` + `secretary_calls`.

## Распределение по нагрузке

Кандидаты — активные менеджеры с заданным добавочным. Нагрузка = число заказов в рабочих
статусах (`status_settings.is_working`). Берётся минимальная нагрузка ниже `max_load`,
тай-брейк — выше `rating`. Если кандидатов нет — перевод на `TELPHIN_FALLBACK_EXTENSION`.

## Настройка (чеклист запуска)

1. Применить миграцию `20260628_telphin_secretary.sql`.
2. Заполнить добавочные менеджеров в `/settings/managers` («Доб. Телфин»).
3. Прописать env: `TELPHIN_SECRETARY_TOKEN`, `TELPHIN_FALLBACK_EXTENSION`, при необходимости
   `TELPHIN_TRANSFER_VAR`/`TELPHIN_TTS_*`.
4. В схеме Телфина: узел «AI секретарь» с классификаторами «текущий заказ» / «новая заявка»;
   в каждой ветке — узел «Интерактивная обработка» с URL
   `https://<домен>/api/telphin/secretary?token=<TOKEN>&mode=existing|new`,
   далее узел «Перевод на номер из переменной» (`transferDestination`).
5. Для ветки existing — узел донабора номера заказа перед «Интерактивной обработкой».

## Журнал

Каждое обращение пишется в `secretary_calls` (вход, решение, исход) — для отладки и отчётности.
