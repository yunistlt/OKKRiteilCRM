# Премия за категории товара (как реализовано)

Статус: **реализовано в проде**.

## Модель

Премия за заявки (`premia_zayavki`) считает только **тип клиента**: новый / постоянный
(по истории оплаченных сделок). Категория товара — отдельные **добавочные** блоки:

- **`premia_categorii`** (group `premia`, умножается на К_качества и К_команды) —
  доплата за заявки заданной категории: режим **«Сумма»** (₽ за заявку) или
  **«% от продажи»** (% от выручки без НДС по заявкам категории).
- **`coef_categorii`** (multiplier, scope `variableBracket`) — множитель всей переменной
  части, если у менеджера есть заявки заданной категории (несколько перемножаются).

Премия за категорию **добавляется** к клиентской премии (не заменяет её).

## Данные

- Категория заказа = `orders.customFields.typ_castomer` (одно значение на заказ; у позиций
  категории нет). Имена категорий — из справочника RetailCRM `kategoriya_klienta`
  (`retailcrm_dictionaries`, синк `/api/sync/dictionaries`).
- Метрики (`lib/salary/metrics.ts`): `countsByType {new, permanent}` (тип клиента),
  `countsByCategory` / `revenueByCategory` (по категории товара).
- В расшифровке (explain) имена категорий подставляются из `ctx.categoryNames`
  (движок грузит их из справочника RetailCRM).

## Конструктор

В «Настройки мотивации → Схемы» админ добавляет блоки «Премия за категории товаров» /
«Коэффициент за категории» и выбирает категории из выпадающего списка (справочник CRM),
задаёт режим и ставку. Никаких кодов/слагов и захардкоженных категорий.

## Файлы

- `lib/salary/blocks/core-blocks.ts` — `premia_zayavki`, `premia_categorii`, `coef_categorii`.
- `lib/salary/metrics.ts` — агрегаты по категориям; `classifyOrderType` (только тип клиента).
- `lib/salary/engine.ts` — `loadCategoryNames` (имена из CRM) → `ctx.categoryNames`.
- `app/salary/settings/ConstructorTabs.tsx` — выбор категорий из справочника, человеческие лейблы.
